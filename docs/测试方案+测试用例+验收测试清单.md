# GPA 管理系统 V1 测试方案 + 测试用例 + 验收测试清单

## 1. 测试目标

本测试文档用于指导 GPA 管理系统 V1 后端在测试阶段的执行工作，目标如下：

- 验证课程管理、成绩管理、GPA 计算、目标规划、批量导入五个已实现模块的功能正确性。
- 验证系统对关键边界条件、异常输入、非法数据和批量事务失败场景的处理是否符合设计。
- 验证规划数据与真实成绩严格隔离，不会污染真实 GPA 结果。
- 验证批量导入具备“先校验后写入、失败整批回滚、成功后 GPA 联动更新”的核心特性。
- 为项目验收提供可直接执行的验收测试清单和结论模板。

## 2. 测试范围

### 2.1 本次测试范围

- 课程管理
  - 新增课程
  - 修改课程
  - 删除课程
  - 查询课程列表

  - 修改成绩
  - 清空成绩
  - 自动换算绩点

  - 计算当前 GPA
  - 统计已计入 GPA 的总学分
  - 统计已计入 GPA 的课程数
  - 返回课程 GPA 明细
- 目标规划
  - 输入目标 GPA
  - 倒推剩余未修课程平均要求
  - 乐观 / 中性 / 保守三种情景模拟
  - 规划数据独立存表，与真实成绩严格隔离

  - 课程导入

## 3. 测试策略

### 3.1 单元测试

目标：验证单个规则或单个服务方法的计算逻辑、校验逻辑和异常处理。

建议重点：

- 绩点规则引擎
  - 百分制转绩点公式是否正确
  - 等级制到绩点映射是否正确
  - 边界值和非法值是否拦截
- 课程服务
  - 学期格式校验
  - 学分正数校验
  - 同学期同名课程去重
  - 状态切换与成绩记录联动
- 成绩服务
  - 未修课程不可录入真实成绩
  - 成绩类型缺失时是否拦截
  - 清空成绩后是否恢复为“已修无成绩”
- GPA 计算服务
  - 只统计已修且有有效成绩课程
  - 加权平均计算是否正确
- 导入服务
  - 文本解析
  - 必填字段校验
  - 非法字段 / 冲突 / 重复识别
  - 整批失败返回结果是否正确
- 规划服务
  - 目标 GPA 合法性校验
  - 所需未来平均绩点倒推是否正确
  - 不可达目标识别
  - 情景覆盖完整性判断

### 3.2 集成测试

目标：验证多个服务、仓储、数据库表之间的协同是否正确。

建议重点：

- 课程新增为“已修”时自动建立空成绩记录。
- 成绩录入后 GPA 汇总立即变化。
- 清空成绩后 GPA 汇总立即回退。
- 更新课程的成绩类型后，已存在成绩的绩点是否重新换算。
- 删除课程后，其成绩记录是否一并删除且 GPA 正确变化。
- 规划结果写入规划表后，真实成绩表和 GPA 汇总不受影响。
- 批量导入课程 / 成绩后，课程表、成绩表、GPA 汇总三者结果一致。

### 3.3 系统测试

目标：按真实使用顺序，以系统视角验证完整业务链路。

建议覆盖的主流程：

- 手工新增课程 -> 录入成绩 -> 查看 GPA 汇总和明细。
- 手工修改课程 / 成绩 -> 查看 GPA 是否同步变化。
- 创建目标 GPA -> 查看所需未来平均绩点 -> 填写三种情景预期 -> 查看模拟结果。
- 批量导入课程 -> 批量导入成绩 -> 验证课程数据、成绩数据、GPA 结果。
- 批量导入非法数据 -> 验证整批回滚且既有数据不受影响。

### 3.4 验收测试

目标：从最终用户角度验证系统 V1 是否满足课程项目交付要求。

验收关注点：

- 核心功能可用
- 主要数据计算正确
- 异常输入可拦截
- 数据一致性可保证
- 规划数据与真实成绩完全隔离
- 批量导入满足“要么全部成功，要么全部失败”

## 4. 测试环境建议

### 4.1 软硬件环境

- 操作系统：Windows 10/11 为主，建议补充一次 macOS 或 Linux 冒烟验证
- Python：3.11 及以上
- 数据库：SQLite
- 运行方式：本地离线运行
- 测试框架：`unittest`

### 4.2 数据环境

建议每轮测试使用独立 SQLite 数据库文件或内存数据库，避免历史数据干扰。

建议准备三类测试数据：

- 基础正常数据
  - 已修课程 2~3 门
  - 未修课程 2~3 门
- 边界数据
  - 百分制 0、59、60、100
  - 等级制 优、良好、中等、及格、不及格
- 异常数据
  - 非法学期
  - 非法学分
  - 非法成绩
  - 导入缺字段、重复字段、冲突字段

### 4.3 执行建议

- 单元 / 集成测试执行：`python -m unittest discover -s tests -p "test_*.py"`
- 手工验收建议按本文第 8 节顺序执行。
- 每次执行批量导入测试前记录导入前课程数、成绩数、当前 GPA，便于验证回滚。

## 5. 功能测试点清单

### 5.1 课程管理

- 新增已修课程，系统创建课程记录并初始化“无成绩”状态。
- 新增未修课程，系统仅创建课程记录，不创建真实成绩记录。
- 新增课程时校验课程名非空、学期非空、学分大于 0。
- 学期格式必须符合“YYYY春/夏/秋/冬”。
- 同一学期下不允许同名课程重复。
- 修改课程基础信息后查询结果正确更新。
- 已修课程改为未修课程时，如已有真实成绩，应禁止直接修改。
- 已修且已有成绩课程修改成绩类型时，应重新换算绩点。
- 删除课程后课程列表不再显示，相关成绩与 GPA 不再计入。

### 5.2 成绩管理

- 已修课程可录入成绩。
- 未修课程不可录入真实成绩。
- 修改成绩后绩点自动重新换算。
- 清空成绩后课程仍保留为已修，但不计入 GPA。
- 百分制与等级制均支持录入。
- 若课程未定义成绩类型，录入时可显式指定成绩类型。
- 非法成绩值必须拦截，不得落库。

### 5.3 GPA 计算

- 当前 GPA 按“学分加权平均”计算。
- 只统计“已修 + 有成绩 + 有成绩类型”的课程。
- 统计已计入 GPA 的总学分正确。
- 统计已计入 GPA 的课程数正确。
- 返回课程 GPA 明细时，包含课程名、学期、学分、原始成绩、绩点、质量点。
- 当没有任何课程计入 GPA 时，当前 GPA 应为空，而不是错误数值。

### 5.4 目标规划

- 可创建目标 GPA 规划记录。
- 系统根据当前 GPA、已修学分、未修学分倒推未来平均绩点要求。
- 自动生成乐观 / 中性 / 保守三种情景。
- 可为未修课程分别录入各情景下的预期成绩。
- 情景未覆盖全部未修课程时，需标识为“未全覆盖”。
- 目标不可达时，系统需返回不可达标识和原因。
- 规划数据仅影响规划结果，不影响真实成绩表与当前 GPA。

### 5.5 批量导入

- 支持逐行 `key=value;...` 文本导入。
- 空行和 `#` 注释行忽略。
- 单行内字段重复时报错。
- 缺少必填字段时报错。
- 存在未知字段时报错。
- 完全重复数据 skip。
- 与已存在数据字段冲突 error。
- 同一批次内部重复且内容一致 skip；内容冲突 error。
- 成绩导入前必须先存在课程。
- 未修课程不可导入真实成绩。
- 课程或成绩批量导入中只要有一条失败，整批不落库。
- 成绩导入成功后 GPA 自动更新。

## 6. 关键边界条件测试

### 6.1 百分制 / 等级制边界值

- 百分制边界：`0`、`59`、`60`、`100`
- 百分制非法：`-1`、`100.1`、`abc`
- 等级制合法：`优`、`良好`、`中等`、`及格`、`不及格`
- 等级制非法：`良`、`通过`、空字符串

重点验证：

- `59` 对应绩点 `0.0000`
- `60` 对应绩点 `1.0000`
- `100` 对应绩点 `4.0000`

### 6.2 未修课程

- 未修课程不得录入真实成绩。
- 未修课程不计入当前 GPA。
- 未修课程只能在目标规划中设置预期成绩。

### 6.3 已修无成绩

- 已修课程刚创建时可处于“无成绩”状态。
- 已修但无成绩课程不应计入 GPA。
- 清空成绩后应恢复到该状态。

### 6.4 非法成绩

- 百分制超范围
- 等级制非法枚举
- 成绩类型缺失
- 成绩与课程成绩类型冲突

预期：全部拦截，不落库，不影响已有 GPA。

### 6.5 目标不可达

- 目标 GPA 大于 4.0
- 当前剩余未修课程不足以把 GPA 提升到目标

预期：

- 非法目标直接报错
- 不可达目标允许生成规划，但必须标识 `feasible = false`，并给出不可达原因

### 6.6 情景未全覆盖

- 仅为部分未修课程填写预期成绩

预期：

- 返回模拟 GPA
- `is_full_coverage = false`
- 已覆盖学分数正确

### 6.7 导入字段缺失 / 非法 / 冲突

- 缺少 `course_name` / `semester` / `credit` / `status`
- 非法 `score_type`
- 未知字段
- 重复字段
- 同一批次冲突
- 与库内现有数据冲突

预期：校验阶段报错，整批不写入。

### 6.8 批量回滚

- 批次中同时存在正确记录与错误记录

预期：

- 整批 `applied = false`
- 成功数为 `0`
- 新数据不落库
- 原有数据保持不变
- 原 GPA 保持不变

## 9. 缺陷优先级建议

| 优先级 | 定义 | 典型示例 |
| --- | --- | --- |
| P0 | 阻塞验收或导致核心数据错误，必须立即修复 | GPA 计算错误；批量回滚失效；规划数据污染真实成绩；系统无法新增 / 导入核心数据 |
| P1 | 主要功能异常，有替代路径但影响较大 | 已修课程无法录入成绩；目标规划结果错误；删除课程后 GPA 未同步 |
| P2 | 一般功能缺陷，不阻塞主流程 | 错误提示不准确；导入报告统计不一致；个别边界值处理不完整 |
| P3 | 轻微问题或文案问题 | 提示文本不规范；展示格式不统一；非关键字段显示异常 |

缺陷提单建议至少包含：

- 缺陷标题
- 模块
- 前置条件
- 复现步骤
- 实际结果
- 预期结果
- 影响范围
- 优先级
- 是否可稳定复现
